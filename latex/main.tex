\begin{abstract}
Prompt2Model turns a free-form request into a runnable vision pipeline. The current prototype covers typed prompt parsing, label resolution, augmentation selection, lightweight model choice for classification and COCO-style detection, smoke-test training, and ONNX export. On a 12-prompt rubric, task and label extraction are 91.7\% exact-match, while priority and environment tags are 100\%; the label resolver is 100\% on an 8-case synonym check. For real-data evaluation, we downloaded Beans and CIFAR-10. On Beans, the prompt-guided low-light recipe improves corrupted-test accuracy from 44.5\% to 62.5\%; on CIFAR-10, the prompt-guided motion-blur recipe improves blur accuracy from 38.7\% to 66.5\%. A Beans ablation also shows that pretrained initialization matters and that EfficientNet-B0 outperforms MobileNetV3-Small under the same recipe.
\end{abstract}

\section{Introduction}
Building a vision model still involves many small engineering choices: task type, class names, augmentation, backbone, and export settings. Prompt2Model uses the prompt as the interface for those choices. In the current system, phrases such as ``prioritize speed'', ``low light'', or ``motion blur'' are translated into an explicit training recipe.

This report captures the week-4 integrated prototype plus real-data extensions. The aim is to show that the pieces work together end to end, with evidence beyond toy smoke outputs.

\section{Method}
The pipeline has three pieces. \textbf{Parsing:} a typed schema stores task type, requested labels, deployment constraints, and environment tags, and a deterministic parser fills that schema from prompt text. \textbf{Label grounding and model choice:} requested labels are matched to dataset classes with a lexical resolver today, while the CLIP path is already wired for later use~\cite{radford2021clip}; the parsed priority signal selects lightweight backbones, currently MobileNetV3 for classification~\cite{howard2019searching} and SSDLite320-MobileNetV3 for detection~\cite{liu2016ssd,howard2019searching}. \textbf{Training and export:} the system loads ImageFolder-like classification data or COCO-style detection annotations, trains a model, evaluates it, and exports an ONNX artifact with embedded metadata. In the current implementation, ``prioritize speed'' selects MobileNetV3-Small, ``low light'' enables brightness/contrast plus blur, and ``motion blur'' enables blur augmentation during training.

\section{Experimental Settings}
To verify integration, we combine repository-level checks with two real-data benchmarks. The validation tracks are a 12-prompt parsing rubric, an 8-case synonym benchmark, a Beans low-light study, a CIFAR-10 motion-blur study, and smoke pipelines for classification and COCO-style detection export. For Beans, we used the official Hugging Face train/validation/test splits (1034/133/128 images). For CIFAR-10, we used a balanced 10k-train/2k-val subset from the standard training split plus the full official test set. Both studies use ImageNet-initialized MobileNetV3-Small and compare a fixed recipe against a prompt-guided corruption-specific recipe. Training ran on Apple Silicon MPS; latency and ONNX measurements were collected on CPU.

\section{Preliminary Results}
Figure~\ref{fig:qualitative} shows the Beans benchmark. The main result is a clear robustness tradeoff: the prompt-guided low-light recipe improves corrupted-test accuracy from 44.5\% to 62.5\%, a gain of 18.0 points, while clean accuracy changes from 94.5\% to 90.6\%. The second benchmark shows the same pattern on CIFAR-10: the motion-blur recipe improves blur accuracy from 38.7\% to 66.5\% with almost no clean penalty (78.5\% to 79.0\%). Table~\ref{tab:ablation_results} shows the Beans ablation. Pretrained initialization matters: the guided MobileNetV3-Small model trained from scratch collapses to 33.6\% on both clean and low-light evaluation. Backbone choice also matters after parsing: guided EfficientNet-B0 reaches 95.3\% clean and 66.4\% low-light accuracy, improving on guided MobileNetV3-Small. The parser and deployment stack are stable enough for these experiments: task and label extraction are both 91.7\% exact-match, label resolution is 100\% on 8 curated synonym cases, both smoke pipelines export to ONNX successfully, and CPU latency is 10.2\,ms for MobileNetV3-Small and 28.7\,ms for SSDLite320.
